# Gmail Watcher

Monitors your Gmail inbox 24/7 and creates actionable markdown files for new emails.

## Setup

### 1. Enable Gmail API

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project or select existing
3. Enable **Gmail API**: `APIs & Services > Library > Gmail API > Enable`
4. Create OAuth2 credentials: `APIs & Services > Credentials > Create Credentials > OAuth client ID`
5. Download `credentials.json` to a secure location

### 2. Install Dependencies

```bash
cd AI_Employee_Vault/watchers
pip install google-api-python-client google-auth-httplib2 google-auth-oauthlib
```

### 3. First-Time Authentication

```bash
python gmail_watcher.py /path/to/vault --authenticate
```

This opens a browser window for OAuth2 consent. Token is saved to `token.json`.

## Configuration

Create `.env` file in watchers directory:

```bash
GMAIL_CLIENT_ID=your_client_id
GMAIL_CLIENT_SECRET=your_client_secret
GMAIL_CREDENTIALS_PATH=/path/to/credentials.json
GMAIL_TOKEN_PATH=/path/to/token.json
GMAIL_CHECK_INTERVAL=120
GMAIL_LABEL_FILTER=IMPORTANT
GMAIL_MAX_RESULTS=10
```

## Usage

### Start Watcher

```bash
# Using default settings (check every 120 seconds)
python gmail_watcher.py D:\personal-AI-empy\AI_Employee_Vault

# Custom check interval (every 60 seconds)
python gmail_watcher.py D:\personal-AI-empy\AI_Employee_Vault --interval 60

# Only monitor specific label (e.g., "Inbox", "Important")
python gmail_watcher.py D:\personal-AI-empy\AI_Employee_Vault --label "Important"

# Fetch more emails per check
python gmail_watcher.py D:\personal-AI-empy\AI_Employee_Vault --max-results 20
```

### Stop Watcher

Press `Ctrl+C` in terminal or:

```bash
taskkill /F /FI "WINDOWTITLE eq Gmail Watcher*"
```

## How It Works

1. **Monitors** Gmail every 120 seconds (configurable)
2. **Filters** for unread + important messages
3. **Detects** priority keywords (urgent, invoice, payment, asap)
4. **Creates** action file in `Needs_Action/` folder
5. **Marks** email as read after processing (optional)

## Action File Format

Generated files follow this structure:

```markdown
---
type: email
from: sender@example.com
subject: Urgent: Project Update Needed
received: 2026-03-13T11:30:00
priority: high
status: pending
message_id: abc123xyz
---

## Email Information

- **From:** sender@example.com
- **Subject:** Urgent: Project Update Needed
- **Received:** 2026-03-13T11:30:00
- **Priority:** HIGH

## Email Content

[Full email body here]

## Suggested Actions

- [ ] Reply to sender
- [ ] Forward to relevant party
- [ ] Archive after processing
- [ ] Create follow-up task

## Notes

*Add your notes here...*

---
*Generated by Gmail Watcher v0.2*
```

## Priority Detection

| Keyword | Priority | Example |
|---------|----------|---------|
| urgent, asap, emergency | high | "Urgent: Need response ASAP" |
| invoice, payment, due | high | "Invoice #1234 Payment Due" |
| deadline, today, tomorrow | medium | "Deadline approaching" |
| meeting, call, schedule | medium | "Meeting request" |
| (default) | medium | Normal emails |

## File Naming Convention

```
EMAIL_<message_id>_<YYYY-MM-DD>.md
```

Example: `EMAIL_18f3a2b1c4d5_2026-03-13.md`

## Integration with Claude Code

Once action files are created, Claude Code can:

1. **Read** all files in `Needs_Action/`
2. **Process** each email (draft replies, categorize)
3. **Create** `Plan.md` for complex tasks
4. **Move** completed items to `Done/`

Example Claude prompt:
```
Process all email action files in Needs_Action folder.
Draft replies for each and save to Pending_Approval if sensitive.
Move processed files to Done folder.
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `ModuleNotFoundError` | Run `pip install google-api-python-client google-auth-httplib2 google-auth-oauthlib` |
| `token.json not found` | Run `--authenticate` flag first |
| `No emails found` | Check if Gmail API is enabled |
| `Duplicate emails` | Delete processed files from `Needs_Action/` |
| `Auth expired` | Re-run authentication with `--authenticate` |
| `API quota exceeded` | Increase check interval or reduce max_results |

## Security Notes

- **Never** commit `token.json` or `credentials.json` to git
- Store credentials in encrypted location
- Token has limited scope (read-only Gmail access by default)
- Revoke access anytime from [Google Account](https://myaccount.google.com/permissions)
