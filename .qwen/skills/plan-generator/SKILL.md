# Plan.md Generator

Creates structured task plans for complex multi-step workflows.

## Overview

When Claude Code detects complex tasks in `Needs_Action/` folder that require multiple steps, it creates a `Plan.md` file in the `Plans/` folder to track progress and ensure nothing is missed.

## Usage

### Manual Plan Creation

```bash
# Create plan for specific task
claude "Create a plan for processing the email in Needs_Action/EMAIL_*.md"

# Create plan for all pending items
claude "Review all files in Needs_Action and create Plan.md for complex tasks"
```

### Automatic Plan Creation

Plans are automatically created when:
1. Watcher creates action file with `priority: high`
2. Task requires 3+ steps to complete
3. Multiple related items need coordination

## Plan.md Template

```markdown
---
type: plan
created: 2026-03-13T11:00:00Z
status: in_progress
priority: high
estimated_duration: 2 hours
related_items:
  - Needs_Action/EMAIL_abc123_2026-03-13.md
  - Needs_Action/FILE_invoice_456_2026-03-13.md
---

# Task Plan: Process Client Invoice Request

## Objective
Respond to client's urgent invoice request and process payment.

## Steps

### Phase 1: Information Gathering
- [ ] Read email from client (EMAIL_abc123)
- [ ] Locate original invoice in Files/Invoices/
- [ ] Verify payment status in Accounting/
- [ ] Check company handbook for payment rules

### Phase 2: Action
- [ ] Draft response email (save to Pending_Approval)
- [ ] Prepare invoice copy
- [ ] Create payment approval request if needed

### Phase 3: Follow-up
- [ ] Send approved response
- [ ] Schedule follow-up reminder
- [ ] Update Dashboard with status

## Dependencies
- Requires approval for payment > $500
- Waiting on accounting system access

## Notes

### Research Findings
*Add findings here*

### Blockers
*List any blockers here*

## Completion Criteria
- [ ] Client receives response
- [ ] Invoice sent
- [ ] Payment scheduled (if approved)
- [ ] All files moved to Done folder

---
*Generated by Plan Generator v0.1*
```

## Plan Status Workflow

```
in_progress → blocked → in_progress → completed
                ↓
            rejected
```

### Status Definitions

| Status | Description |
|--------|-------------|
| `pending` | Plan created but not started |
| `in_progress` | Currently working on steps |
| `blocked` | Waiting on external factor |
| `completed` | All steps done, moved to Done |
| `rejected` | Plan cancelled, moved to Rejected |

## Integration with Claude Code

### Reading Plans

Claude reads plans to understand:
1. What steps are completed
2. What steps are next
3. What blockers exist

### Updating Plans

After each action, Claude updates the plan:
1. Check off completed steps
2. Add notes with outcomes
3. Update status if blocked or complete

### Example Claude Prompt

```
Review Plan.md in Plans/ folder.
Complete the next unchecked step.
Update the plan with progress.
If all steps complete, move to Done folder.
```

## Plan Types

### Email Response Plan
For complex email conversations requiring research.

### Payment Processing Plan
For invoices, payments, and financial approvals.

### Project Launch Plan
For new projects with multiple milestones.

### Crisis Management Plan
For urgent issues requiring immediate action.

## Best Practices

| Practice | Description |
|----------|-------------|
| **One plan per complex task** | Don't combine unrelated tasks |
| **Clear completion criteria** | Define what "done" looks like |
| **Time estimates** | Add estimated duration for each phase |
| **Dependencies noted** | List what you're waiting on |
| **Regular updates** | Update plan after each step |

## File Naming Convention

```
Plan_<TaskType>_<Date>_<ID>.md
```

Examples:
- `Plan_EmailResponse_2026-03-13_001.md`
- `Plan_PaymentProcessing_2026-03-13_002.md`
- `Plan_ProjectLaunch_2026-03-14_001.md`

## Folder Structure

```
Plans/
├── active/           # Currently in progress
│   └── Plan_*.md
├── completed/        # Finished plans
│   └── Plan_*.md
└── archive/          # Old plans (quarterly archive)
    └── 2026-Q1/
```

```markdown
| Client Invoice #1234 | High | 60% | In Progress |

```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Plan not updated | Set reminder to update after each step |
| Too many active plans | Consolidate related plans |
| Plans never completed | Break into smaller steps |
| Steps too vague | Make each step specific and actionable |
