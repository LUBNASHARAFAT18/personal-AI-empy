# LinkedIn Auto-Poster

Automatically generates and posts business updates to LinkedIn to generate sales and engagement.

## ⚠️ Important Notes

- **Terms of Service**: Using automated tools with LinkedIn may violate their Terms of Service. Use at your own risk.
- **Rate Limiting**: Post frequency should be reasonable (1-3 times per day maximum)
- **Session Security**: LinkedIn session data is stored locally. Never share your session folder.
- **Approval Required**: All posts require human approval before publishing (HITL pattern)

## Setup

### 1. Install Dependencies

```bash
cd AI_Employee_Vault/watchers
pip install playwright
playwright install chromium
```

### 2. First-Time Login

```bash
python linkedin_poster.py /path/to/AI_Employee_Vault --login
```

This opens a browser window. Log in to LinkedIn normally. Session is saved for future use.

## Configuration

Create `.env` file in watchers directory:

```bash
LINKEDIN_SESSION_PATH=./linkedin_session
LINKEDIN_POST_SCHEDULE=09:00,15:00
LINKEDIN_MAX_POSTS_PER_DAY=3
LINKEDIN_HEADLESS=true
```

## Usage

### Create Post (Manual)

```bash
# Create a post from text
python linkedin_poster.py D:\personal-AI-empy\AI_Employee_Vault --create-post "Excited to announce our new AI Employee service!"

# Create post from file (markdown supported)
python linkedin_poster.py D:\personal-AI-empy\AI_Employee_Vault --create-post-file posts/draft_post.md

# Create post with hashtags
python linkedin_poster.py D:\personal-AI-empy\AI_Employee_Vault --create-post "New milestone reached!" --hashtags "#AI #Automation #Business"
```

### Post to LinkedIn

```bash
# Post with approval workflow (creates Pending_Approval file)
python linkedin_poster.py D:\personal-AI-empy\AI_Employee_Vault --post --approval-required

# Post directly (use with caution)
python linkedin_poster.py D:\personal-AI-empy\AI_Employee_Vault --post
```

### Scheduled Posting

```bash
# Start scheduler (runs in background)
python linkedin_poster.py D:\personal-AI-empy\AI_Employee_Vault --schedule

# Schedule with custom times
python linkedin_poster.py D:\personal-AI-empy\AI_Employee_Vault --schedule --schedule-times "09:00,15:00,18:00"
```

## How It Works

1. **Content Generation**: AI generates post content based on business goals, recent wins, or topics
2. **Approval File**: Creates `PENDING_APPROVAL_LinkedIn_Post_*.md` in `Pending_Approval/` folder
3. **Human Review**: User reviews and moves file to `Approved/` or `Rejected/`
4. **Auto-Post**: Watcher monitors `Approved/` folder and posts via browser automation
5. **Confirmation**: Moves post to `Done/` with confirmation screenshot

## Approval File Format

```markdown
---
type: approval_request
action: linkedin_post
content: "Excited to share our Q1 results..."
hashtags: "#AI #Automation #Business"
scheduled_time: 2026-03-13T15:00:00
created: 2026-03-13T10:00:00
status: pending
---

## LinkedIn Post Preview

**Content:**
Excited to share our Q1 results! Our AI Employee service has helped 50+ businesses automate their workflows. 🚀

**Hashtags:**
#AI #Automation #Business #Productivity

**Scheduled For:**
2026-03-13 at 3:00 PM

## To Approve
1. Review the content above
2. Move this file to `Pending_Approval/` folder
3. AI will post automatically when approved

## To Reject
Move this file to `Rejected/` folder with reason

---
*Generated by LinkedIn Poster v0.1*
```

## Posting Best Practices

| Aspect | Recommendation |
|--------|----------------|
| **Frequency** | 1-3 posts per day maximum |
| **Timing** | 9 AM, 3 PM, 6 PM (highest engagement) |
| **Length** | 100-300 characters (optimal) |
| **Hashtags** | 3-5 relevant hashtags |
| **Media** | Include images when possible |
| **Engagement** | Respond to comments within 24 hours |

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `Session not found` | Run `--login` to authenticate |
| `Post failed` | Check internet, re-run `--login` |
| `LinkedIn detected automation` | Reduce posting frequency, use longer intervals |
| `Browser crashes` | Delete `linkedin_session/` and re-login |
| `Character limit exceeded` | LinkedIn allows 3000 chars, but shorter is better |

## Security Notes

- **Never** commit `linkedin_session/` folder to git
- Store session data in encrypted location
- Use approval workflow for all posts
- Session expires periodically - re-authenticate when needed
