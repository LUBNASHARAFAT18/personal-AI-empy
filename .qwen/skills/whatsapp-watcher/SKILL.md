# WhatsApp Watcher

Monitors WhatsApp Web 24/7 and creates actionable markdown files for urgent messages.

## ⚠️ Important Notes

- **Terms of Service**: Using automated tools with WhatsApp may violate WhatsApp's Terms of Service. Use at your own risk.
- **Session Security**: WhatsApp session data is stored locally. Never share your session folder.
- **Rate Limiting**: Keep check intervals reasonable (30+ seconds) to avoid detection.

## Setup

### 1. Install Dependencies

```bash
cd AI_Employee_Vault/watchers
pip install playwright
playwright install chromium
```

### 2. First-Time QR Code Scan

```bash
python whatsapp_watcher.py /path/to/AI_Employee_Vault --setup-session
```

This opens a browser window showing WhatsApp QR code. Scan with your phone to authenticate.

Session is saved to `watchers/whatsapp_session/`.

## Configuration

Create `.env` file in watchers directory:

```bash
WHATSAPP_CHECK_INTERVAL=30
WHATSAPP_SESSION_PATH=./whatsapp_session
WHATSAPP_KEYWORDS=urgent,asap,invoice,payment,help,emergency
WHATSAPP_HEADLESS=false
```

## Usage

### Start Watcher

```bash
# Using default settings (check every 30 seconds)
python whatsapp_watcher.py D:\personal-AI-empy\AI_Employee_Vault

# Custom check interval (every 60 seconds)
python whatsapp_watcher.py D:\personal-AI-empy\AI_Employee_Vault --interval 60

# Custom keywords
python whatsapp_watcher.py D:\personal-AI-empy\AI_Employee_Vault --keywords "urgent,payment,invoice,asap"

# Run with visible browser (debugging)
python whatsapp_watcher.py D:\personal-AI-empy\AI_Employee_Vault --headless false
```

### Stop Watcher

Press `Ctrl+C` in terminal or:

```bash
taskkill /F /FI "WINDOWTITLE eq WhatsApp Watcher*"
```

## How It Works

1. **Launches** Chromium browser with saved WhatsApp session
2. **Navigates** to WhatsApp Web
3. **Scans** chat list for unread messages
4. **Filters** messages containing urgent keywords
5. **Creates** action file in `Needs_Action/` folder

## Action File Format

Generated files follow this structure:

```markdown
---
type: whatsapp
from: +1234567890
chat: John Doe
received: 2026-03-13T11:30:00
priority: high
status: pending
keywords: urgent, invoice
---

## Message Content

**John Doe:** Hey, urgent! Need the invoice for project X ASAP. Client is asking.

## Context

- **Chat:** John Doe
- **Keywords Detected:** urgent, invoice
- **Timestamp:** 2026-03-13T11:30:00

## Suggested Actions

- [ ] Read message and understand urgency
- [ ] Check if invoice exists
- [ ] Draft response (save to Pending_Approval)
- [ ] Send invoice if available
- [ ] Follow up if action needed

## Notes

*Add your notes here...*

---
*Generated by WhatsApp Watcher v0.1*
```

## Priority Detection

| Keyword | Priority | Example |
|---------|----------|---------|
| urgent, asap, emergency | high | "Urgent! Need help" |
| invoice, payment, money | high | "Where is my invoice?" |
| deadline, today, tomorrow | medium | "Deadline today" |
| help, question, need | medium | "Need your help" |
| (default) | low | Normal messages |

## File Naming Convention

```
WHATSAPP_<chat_name>_<timestamp>_<YYYY-MM-DD>.md
```

Example: `WHATSAPP_JohnDoe_1710334200_2026-03-13.md`

## Integration with Claude Code

Once action files are created, Claude Code can:

1. **Read** all files in `Needs_Action/`
2. **Process** each message (draft replies, take action)
3. **Create** `Plan.md` for complex tasks
4. **Move** completed items to `Done/`

Example Claude prompt:
```
Process all WhatsApp action files in Needs_Action folder.
Draft replies for urgent messages and save to Pending_Approval.
Move processed files to Done folder.
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `playwright not found` | Run `pip install playwright && playwright install chromium` |
| `Session not found` | Run `--setup-session` to scan QR code |
| `WhatsApp Web not loading` | Check internet connection, try `--headless false` |
| `No messages detected` | Ensure unread messages exist in scanned chats |
| `Browser crashes` | Delete `whatsapp_session/` folder and re-authenticate |
| `QR code expired` | Re-run `--setup-session` and scan quickly |

## Security Notes

- **Never** commit `whatsapp_session/` folder to git
- Store session data in encrypted location
- Watcher only reads messages (no sending capability)
- Session expires periodically - re-authenticate when needed
