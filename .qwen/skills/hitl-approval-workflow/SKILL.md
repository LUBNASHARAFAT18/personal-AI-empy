# HITL Approval Workflow

Human-in-the-Loop pattern for sensitive action approval.

## Overview

For sensitive actions (payments, sending messages, posting content), the AI Employee **never** acts autonomously. Instead, it creates an approval request file that requires human review before execution.

## Approval Workflow

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  AI Creates     │────▶│  Human Reviews   │────▶│  Move to        │
│  Approval File  │     │  & Decides       │     │  Approved/      │
│  (Pending)      │     │                  │     │  or Rejected/   │
└─────────────────┘     └──────────────────┘     └─────────────────┘
                                                        │
                                                        ▼
                                               ┌─────────────────┐
                                               │  AI Executes    │
                                               │  & Moves to Done│
                                               └─────────────────┘
```

## Sensitive Actions Requiring Approval

| Action Type | Approval Required | Auto-Execute |
|-------------|-------------------|--------------|
| **Payments** | Always | Never |
| **Email to new contacts** | Always | Never |
| **Email replies (sensitive)** | Always | Never |
| **Social media posts** | Always | Never |
| **WhatsApp messages** | Always | Never |
| **File sharing** | Always | Never |
| **Subscription changes** | Always | Never |
| **Routine emails (known contacts)** | Optional | If approved |
| **File organization** | No | Yes |

## Approval File Format

```markdown
---
type: approval_request
action: payment
amount: 500.00
recipient: Client ABC Corp
reason: Invoice #1234 payment
created: 2026-03-13T10:30:00Z
expires: 2026-03-14T10:30:00Z
status: pending
---

# Approval Required: Payment

## Details

- **Action:** Payment
- **Amount:** $500.00
- **Recipient:** Client ABC Corp
- **Reason:** Invoice #1234 payment
- **Due Date:** 2026-03-20

## Context

This payment is for services rendered in February 2026.
Invoice has been verified and matches our records.

## To Approve

**Option 1: Move to Approved folder**
- Drag this file to `Approved/` folder
- AI will execute automatically

**Option 2: Reject**
- Move to `Rejected/` folder
- Add reason in notes below

## Notes

*Add comments or rejection reason here...*

---
*Generated by HITL Approval Workflow v0.1*
```

## Folder Structure

```
Vault/
├── Pending_Approval/      # Awaiting human decision
│   ├── APPROVAL_REQUIRED_Payment_ClientA.md
│   └── APPROVAL_REQUIRED_Email_Reply.md
├── Approved/              # Approved actions (AI executes)
│   └── (files moved here trigger execution)
├── Rejected/              # Rejected actions
│   └── (files with rejection reason)
└── Done/                  # Completed actions
    └── (executed approvals moved here)
```

## Usage

### Create Approval Request

```python
from hitl_approval import HITLApproval

approval = HITLApproval(vault_path)

# Create payment approval
approval.create_approval(
    action_type='payment',
    details={
        'amount': 500.00,
        'recipient': 'Client ABC',
        'reason': 'Invoice #1234'
    },
    description='Payment for Invoice #1234'
)

# Create email approval
approval.create_approval(
    action_type='email_send',
    details={
        'to': 'client@example.com',
        'subject': 'Re: Project Update',
        'body': 'Dear Client,...'
    },
    description='Reply to client inquiry'
)
```

### Check for Approved Actions

```bash
# Python
python hitl_approval.py /path/to/vault --check-approved

# Or use watcher (runs continuously)
python hitl_approval.py /path/to/vault --watch
```

### Execute Approved Action

```python
# Get approved files
approved = approval.get_approved_actions()

for action_file in approved:
    # Execute based on type
    if action_file.action_type == 'payment':
        execute_payment(action_file.details)
    elif action_file.action_type == 'email_send':
        send_email(action_file.details)
    
    # Move to Done
    action_file.move_to_done()
```

## Human Workflow

### As a Human Reviewer:

1. **Check Pending_Approval folder** regularly
2. **Review** each approval request
3. **Decide:**
   - ✅ **Approve:** Move file to `Approved/` folder
   - ❌ **Reject:** Move to `Rejected/` with reason
4. **Monitor Done folder** for completed actions

### Best Practices

| Practice | Description |
|----------|-------------|
| **Review daily** | Check Pending_Approval at least once per day |
| **Add notes** | Document reasoning for rejections |
| **Batch approvals** | Review multiple approvals together |
| **Set expiry** | Time-sensitive approvals should have expiry |

## AI Workflow

### As AI Employee:

1. **Detect** sensitive action needed
2. **Create** approval file in `Pending_Approval/`
3. **Wait** for human decision
4. **Monitor** `Approved/` folder for execution
5. **Execute** approved actions
6. **Move** to `Done/` with outcome

## Expiry Handling

Approvals can have expiry times:

```yaml
expires: 2026-03-14T10:30:00Z
```

If expired:
1. AI moves file to `Rejected/` with note
2. Creates notification in Dashboard
3. Logs expiry event

## Security Rules

| Rule | Enforcement |
|------|-------------|
| **Never auto-execute** | AI cannot bypass approval |
| **Log all actions** | Every approval logged |
| **Expiry required** | Time-sensitive actions expire |
| **Audit trail** | All decisions recorded |

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Approval not executed | Check if file is in Approved/ folder |
| Wrong action executed | Review approval file details |
| Approval expired | Create new approval request |
| File stuck in Pending | Human needs to review |
